# AI-Use Declaration - Part 2 (Group BB)

Same declaration approach as Part 1 (`part1/ai_use_declaration.md`); this
file covers the Part 2 materials specifically. If the report combines both,
the two declarations should be merged into a single appendix.

**Which tools were used:** Claude (Anthropic), via the Claude Code CLI.

**What they were used for:**

- drafting `detector.py`, `simulate_lockout.py`, and `authlog_parsing.py`,
  including adapting the parsing logic from our Part 1 script
- drafting the fail2ban, sshd, and rsyslog configuration files and their
  comments
- drafting this folder's README and the Part 2 report section
- suggesting the three-layer structure (prevent / detect / preserve) for the
  response

**How the outputs were checked, modified, or built upon:**

- Every quantitative claim in the Part 2 draft (ban times, blocked counts,
  alert counts, lead times, threshold gaps) comes from actually running
  `detector.py` and `simulate_lockout.py` against our assigned extract; the
  outputs are committed under `part2/output/` and can be regenerated with
  the commands in the README.
- Detection thresholds were not taken on trust: the legitimate-vs-campaign
  gaps quoted in the README (max legitimate burst 9 vs minimum campaign
  burst 348, etc.) were measured directly from the extract before the
  thresholds were fixed.
- The R1 detection rule was cross-checked against Part 1: it reproduces the
  117-attempt `deploy` finding exactly, using the same burst logic.
- Configuration recommendations were checked against the evidence (e.g.
  `PermitRootLogin no` was only proposed after confirming the extract
  contains zero legitimate root SSH logins).

**Which parts were substantially shaped by AI:** the initial drafts of all
Part 2 code, configuration, and prose were AI-generated, then reviewed,
tested, and edited by the group.

**How we ensured final claims are supported by Python-verified evidence:**
no number appears in the Part 2 report section that cannot be found in
`part2/output/alerts.csv`, `part2/output/lockout_simulation.csv`, or
regenerated by rerunning the two scripts with their default settings.
